# 人脸识别能力调研报告

## 调研日期
2026-03-04

## 一、微信官方人脸识别能力

### 1. wx.startFacialRecognitionVerify（人脸核身）
- **能力名称**: 微信人脸核身（实名认证级）
- **接入条件**:
  - 需要小程序完成企业主体认证
  - 需要申请开通「人脸核身」接口权限
  - 仅限特定类目（金融、政务、医疗、交通等）可申请
  - 需提供业务场景说明并通过审核
- **是否免费**: ❌ **不免费**
  - 按调用次数收费，通常 0.5~1.0 元/次
  - 需签署服务协议并预充值
- **适用场景**: 实名认证、金融开户等强身份核验场景
- **限制**:
  - 工厂记账类应用不属于允许的类目范围
  - 普通企业主体大概率无法通过类目审核
  - 即使通过审核，按次计费的成本对高频打卡场景不可接受

### 2. 微信小程序相机组件 + 自建人脸比对
- **方案**: 使用 `<camera>` 组件拍照 + 后端人脸比对
- **问题**:
  - 需自建或接入第三方人脸识别服务（如腾讯云人脸识别）
  - 腾讯云人脸识别：免费额度有限（每月1万次），超出后按次计费
  - 需存储人脸特征数据，涉及隐私合规风险
  - 无活体检测能力，容易被照片欺骗

## 三、结论与建议

### 结论
**微信小程序目前不存在可用于「登录+考勤打卡」的官方免费人脸识别能力。**

所有官方人脸核身能力均需付费且有严格的类目限制，工厂记账场景不在允许范围内。

### 实施方案
采用 **Feature Flag（功能开关）** 方式实现：

1. **默认关闭人脸识别功能**
2. 在系统设置中提供开关，开启时显示成本风险提示
3. 预留接口，未来如有免费能力可快速接入
4. 开关关闭时：
   - 登录 = 用户名 + 密码
   - 打卡 = 定位校验通过即可
5. 开关开启时（需管理员确认费用后）：
   - 可接入第三方人脸服务
   - 登录与打卡叠加人脸校验

### 替代方案（推荐）
- 强化密码安全（强制修改默认密码、密码强度校验）
- 定位围栏校验（已实现）
- 管理员操作审计日志（已实现）
- 设备绑定（openid自动绑定，已实现）

## 四、工程实现

已在系统设置中增加「人脸识别开关」（`face_recognition_enabled`），默认值为 `false`。
开关开启时，系统设置页面显示费用风险提示文案。
